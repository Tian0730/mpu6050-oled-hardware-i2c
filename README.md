我预想的4种小车电控方案：
方案1：直行黑线使用灰度位置环PID，转弯灰度只负责判断，由陀螺仪角度环PID负责转精确的90°，之后切换回灰度巡线；检测到白线使用陀螺仪保持航向角。用了2个单环控制
方案2：全程灰度仅负责识别转弯，直行和转弯全靠陀螺仪的航向角校准，即只有角度环
方案3：黑线部分无论直行还是转弯全靠灰度的位置环PID，白线部分依靠陀螺仪的角度环PID保持航向角。用了2个单环控制
方案4：全程均调用陀螺仪的角度环监测航向角，黑线直行还会使用灰度巡线，灰度还负责识别转弯，仅空白部分不调用灰度。用了双环控制，空白为单环

4个方案对比分析
先快速梳理四个方案的核心差异：
          直行黑线	                  转弯	                白线/空白	            控制结构
方案1	  灰度位置环PID	    灰度识别+陀螺仪角度环PID	    陀螺仪航向角保持	      2个单环，分时切换
方案2	  陀螺仪角度环	      陀螺仪角度环	                陀螺仪角度环	          仅1个单环
方案3	  灰度位置环PID	    灰度位置环PID	              陀螺仪角度环PID	      2个单环，分时切换
方案4	  灰度+陀螺仪双环	  灰度识别+陀螺仪角度环	        仅陀螺仪角度环	        双环 ↔ 单环过渡

逐方案分析
方案1 —— 分工明确，但切换有风险
✅ 灰度巡线直行很成熟，位置环PID调好后直线稳定性好
✅ 陀螺仪转90°比纯灰度转弯精度高得多，不受地面反光、线宽变化影响
❌ 模式切换是最大隐患：直行→转弯→直行，两次切换，切换瞬间如果车体姿态没对齐，容易"切飞"
❌ 白线区域只有陀螺仪保持航向角，但对陀螺仪来说没有绝对位置参考，累计漂移会导致越走越偏

方案2 —— 最简但最不可靠
✅ 实现最简单，全程只有一个角度环
❌ 致命缺陷：陀螺仪无法感知横向位置偏移。如果车体被颠簸或碰撞偏离了黑线，陀螺仪完全不知道，车会越跑越偏。陀螺仪只能保证"朝向对"，不能保证"位置对"
❌ 灰度只用来识别转弯，严重浪费了灰度传感器的巡线能力

方案3 —— 灰度扛全场，转弯是短板
✅ 黑线部分全程灰度PID，逻辑简单，没有模式切换
❌ 灰度转弯精度差：灰度传感器视野有限，转弯时黑线容易跑出传感器范围，导致PID失控
❌ 同样存在白线区域陀螺仪漂移问题

方案4 —— 双环融合，兼顾精度与鲁棒性 ⭐
✅ 黑线直行双环控制：灰度位置环保证车不偏离黑线，陀螺仪角度环抑制晃动，两个环互补
✅ 陀螺仪全程在线：不存在"冷启动"问题，从黑线进入白线时陀螺仪已经在稳定跟踪航向角，过渡平滑
✅ 转弯用陀螺仪：继承了方案1转弯精度高的优点
✅ 白线区域单环（陀螺仪），虽然同样没有绝对位置参考，但因为刚从黑线出来时车体姿态是正确的，短距离白线区域陀螺仪漂移影响很小

🏆 我们最终选择了方案4
理由总结：

维度	                        方案4表现
稳定性	        双环互补，黑线直行最稳；转弯陀螺仪精度高
实现难度	      中等——需要调两套PID并做融合，但比方案1的模式切换逻辑简单
过渡平滑性	    陀螺仪全程运行，模式切换时角度环始终在线，没有"接力掉棒"问题
抗干扰	        灰度短时丢线时陀螺仪能兜底，陀螺仪漂移时灰度能校正

✅实现思路：
核心思路是：灰度位置环输出 + 陀螺仪角度环输出 → 加权/叠加后给电机差速。

最终差速 = K1 × 灰度PID输出 + K2 × 陀螺仪PID输出

黑线直行：K1 > 0, K2 > 0  （双环）
转弯阶段：K1 = 0, K2 > 0  （纯陀螺仪转90°）
白线区域：K1 = 0, K2 > 0  （陀螺仪保持航向角）


✅关键点：
转弯时记录当前航向角作为目标角（+90°或-90°），陀螺仪角度环追目标角
转弯完成后，用当前实际航向角更新灰度直行的目标航向角基准，避免切换时角度突变
白线区域 K2 可以适当减小，防止过度修正引起震荡
